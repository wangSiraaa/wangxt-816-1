# 分拣格口压力图看板

## 功能简介

这是一个**可操作的交互式看板**，而非静态图片。提供分拣中心格口压力状态的实时可视化展示与管理功能。

### 核心功能

- **格口状态展示**：32个格口以卡片形式展示，通过颜色区分不同压力等级
- **班次切换**：支持早班、中班、晚班三个班次独立切换与数据管理
- **压力排行**：TOP20 压力排行榜，快速定位高负荷格口
- **全屏看板**：一键进入全屏模式，适用于监控大屏展示
- **格口详情**：点击格口查看详细信息，支持标记堵塞/解除堵塞、投递包裹操作

## 静态页面启动方式

### 方式一：直接打开
```bash
# 直接在浏览器中打开 index.html 文件
open index.html
```

### 方式二：本地 HTTP 服务器（推荐）
```bash
# 使用 Python 内置 HTTP 服务器
python3 -m http.server 20116
```
然后在浏览器中访问：`http://localhost:20116`

## 样例数据说明

### 数据规模
- **班次数量**：3 个（早班、中班、晚班）
- **每班格口数**：32 个
- **总格口数**：96 个
- **目的地城市**：32 个（覆盖全国主要城市）

### 压力等级阈值
基于待分拣数量占最大容量（100件）的百分比划分：

| 等级 | 阈值范围 | 颜色标识 | 说明 |
|------|----------|----------|------|
| 正常 | 0% - 30% | 绿色 | 负荷充足 |
| 轻度 | 30% - 50% | 蓝色 | 正常负荷 |
| 预警 | 50% - 70% | 黄色 | 需要关注 |
| 高压 | 70% - 85% | 橙色 | 高负荷状态 |
| 严重 | 85% - 100% | 红色 | 接近满负荷 |
| 堵塞 | - | 深灰色 | 已标记堵塞 |

## 检查步骤

### 验证标记格口堵塞后投递按钮不可用且刷新不丢失

**步骤 1：启动应用**
- 使用上述任一方式启动应用
- 确认页面正常加载，显示 32 个格口卡片

**步骤 2：点击任意格口打开详情弹窗**
- 点击任意一个格口卡片
- 确认弹出详情弹窗，显示格口编号、目的地、压力等级等信息

**步骤 3：点击【标记堵塞】按钮**
- 在详情弹窗中点击红色的【标记堵塞】按钮
- 确认弹窗中按钮状态变化：【标记堵塞】按钮隐藏，【解除堵塞】按钮显示

**步骤 4：验证【投递包裹】按钮变为禁用状态**
- 确认【投递包裹】按钮变为灰色，不可点击
- 点击该按钮确认无响应

**步骤 5：刷新页面，验证堵塞状态仍然保留**
- 按 F5 或浏览器刷新按钮刷新页面
- 再次点击同一个格口打开详情
- 确认堵塞状态仍然保持（显示【解除堵塞】按钮，投递按钮禁用）
- 确认首页该格口卡片显示为"已堵塞"状态

**步骤 6：点击【解除堵塞】恢复正常**
- 点击【解除堵塞】按钮
- 确认【标记堵塞】按钮重新显示
- 确认【投递包裹】按钮恢复可用状态

**步骤 7：切换班次验证各班次数据独立**
- 在顶部班次选择器中切换到其他班次
- 确认各班次的格口状态独立，互不影响
- 可在不同班次中分别标记堵塞，验证数据隔离

## 项目结构说明

```
.
├── index.html          # 主页面 HTML 文件
├── styles.css          # 样式文件
├── script.js           # 核心业务逻辑 JavaScript
├── smoke.sh            # Smoke 测试脚本
```

### 文件说明

- **index.html**：页面结构定义，包含看板布局、弹窗、按钮等 DOM 元素
- **styles.css**：深色主题样式，包含压力等级颜色、卡片样式、动画效果
- **script.js**：核心业务逻辑，包含数据生成、状态管理、本地存储、事件处理
- **smoke.sh**：自动化冒烟测试脚本，用于验证核心功能完整性
